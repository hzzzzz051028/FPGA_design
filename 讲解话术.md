# 课程设计讲解话术（20 条基本整数指令 + 12 条扩展整数指令版）

## 一、开场话术
大家好，我这次做的是一个基于 Verilog 的单周期 MIPS 模型机课程设计演示版本。  
和通用 demo 模板相比，我没有直接照搬默认参数，而是重新设计了一套测试数据和控制流，完整覆盖 **20 条 MIPS 基本整数指令** 和 **12 条 MIPS 扩展整数指令**。  
这样做的目的有两个：
1. 让测试程序和常见模板明显不同；
2. 保证整个执行流程稳定，不会出现跳飞、死循环或者读未定义值的问题。

我这次覆盖的扩展整数指令一共 12 条：
- `slt`
- `bgtz`
- `bltz`
- `jalr`
- `mult`
- `multu`
- `div`
- `divu`
- `mfhi`
- `mflo`
- `mthi`
- `mtlo`

## 二、总体结构话术
整个模型机我可以按 5 个部分来讲：
1. **IF 取指**：负责维护 `PC` 并从指令存储器取指；
2. **译码和寄存器堆**：把指令拆成 `op/rs/rt/rd/shamt/imm`，同时读寄存器；
3. **EX 执行单元**：完成算术、逻辑、移位、访存地址计算和 HI/LO 控制；
4. **DataMem 数据存储器**：处理 `lw` 和 `sw`；
5. **WB 写回**：把 ALU 结果、访存结果或者跳转链接地址写回寄存器。

如果老师问“这次你主要改了什么”，我可以直接回答：
- 我补齐了 `beq`、`bne`、`bltz`、`bgtz`、`j` 的控制流；
- 我修正了 `andi`、`ori`、`xori` 的零扩展；
- 我修正了 `lw` 的有效地址计算；
- 我把 `20 + 12` 条整数指令的测试程序重新排布成一个稳定的前向执行流；
- 我把寄存器堆 reset 时清零，避免仿真里出现 `X`；
- 我把测试平台从 `$stop` 改成自动 `$finish`，这样仿真可以自动结束。

## 三、程序总体思路话术
我这次的测试程序不是把所有指令随便堆在一起，而是按功能分成 7 段：
1. **基础算术逻辑段**：验证 `addi/add/sub/and/or/xor`；
2. **移位段**：验证 `sll/srl/sra`；
3. **控制转移段一**：用 `jr` 跳过一个 `nop`；
4. **立即数和访存段**：验证 `lui/ori/andi/xori/sw/lw`；
5. **条件分支扩展段**：验证 `beq/bne/bltz/bgtz`；
6. **HI/LO 扩展段**：验证 `mult/multu/div/divu/mfhi/mflo/mthi/mtlo`；
7. **控制转移段二**：验证 `j/jal/jalr`。

整个程序的核心思路是：
- 所有跳转目标都指向后续有效指令；
- 所有分支都只跨过 `nop`；
- 访存使用非零基址加偏移，证明地址计算是真正生效的；
- HI/LO 相关指令按“先写 HI/LO，再读回验证”的顺序安排；
- 仿真跑固定拍数后自动结束。

## 四、模型机指令执行流程话术
如果老师问“模型机一条指令是怎么执行的”，我可以按下面这个顺序回答：

### 1. 取指
首先由 `IF` 模块维护程序计数器 `PC`。  
每来一个时钟上升沿，`IF` 会根据当前控制信号决定下一条地址是顺序加 `4`，还是跳转目标地址。  
然后 `InstMem` 根据 `PC` 输出当前指令，也就是 `instruction`。

### 2. 译码
拿到指令后，顶层模块会把指令拆分成几个字段：
- `op`
- `rs`
- `rt`
- `rd`
- `shamt`
- `imm`

这一步的作用是判断当前是算术指令、逻辑指令、访存指令、分支跳转指令，还是 HI/LO 扩展指令。

### 3. 读寄存器
译码完成以后，`RegFile` 根据 `rs` 和 `rt` 读出两个源操作数：
- `regaData`
- `regbData`

这两个数据会送到执行单元。  
如果当前指令是 `jr` 或 `jalr`，那么后面跳转地址通常就直接来自 `regaData`。

### 4. 执行 / 地址计算 / HI/LO 控制
接下来进入 `EX` 模块：
- 算术类指令如 `add`、`sub` 在这里完成；
- 逻辑类指令如 `and`、`or`、`xor`、`andi`、`ori`、`xori` 在这里完成；
- 移位类指令如 `sll`、`srl`、`sra` 也在这里完成；
- `lw`、`sw` 在这里计算有效地址，也就是 `基址 + 偏移`；
- `mult`、`multu`、`div`、`divu`、`mthi`、`mtlo` 在这里控制 `HI/LO`。

### 5. 访存
如果当前是 `lw` 或 `sw`，那么 `EX` 输出的地址会送到 `DataMem`：
- `sw`：把寄存器数据写入数据存储器；
- `lw`：从数据存储器把数据读出来。

### 6. 写回
执行结果最终要写回寄存器堆。  
本工程的写回数据主要有三种来源：
1. 普通运算结果，来自 `exOut`；
2. `lw` 的读数，来自 `memOut`；
3. `jal/jalr` 的链接地址，来自当前 `pc`。

而 `mfhi/mflo` 也是通过 `exOut` 把 `HI/LO` 的值写回通用寄存器。

### 7. 更新 PC，进入下一条指令
最后再根据指令类型更新 `PC`：
- 普通指令：`PC = PC + 4`
- `beq/bne/bltz/bgtz`：条件成立时跳到分支目标地址
- `j/jal`：跳到指令给出的绝对地址
- `jr/jalr`：跳到寄存器给出的目标地址

### 8. 一句话总结版本
如果答辩时间很紧，我会直接总结成一句话：  
**模型机执行一条指令的过程，就是 `PC 取指 -> 指令译码 -> 读寄存器 -> EX 运算/地址计算/HI/LO控制 -> DataMem 访存 -> 写回寄存器 -> 更新 PC`。**

## 五、20 条基本整数指令讲解话术

### 1. `addi`
我先用 `addi` 构造初始常量，比如 `$1 = 48`、`$2 = 13`、`$17 = 36`。  
这样既能给后面的算术运算提供操作数，也能给 `jr` 和访存提供目标地址或基址。

### 2. `add`
`add $3,$1,$2` 用来验证最基本的寄存器加法。  
这里结果是 `48 + 13 = 61`，也就是 `$3 = 0x3D`。

### 3. `sub`
`sub $4,$1,$2` 用来验证减法通路。  
结果是 `35`，也就是 `$4 = 0x23`。

### 4. `and`
`and $5,$1,$2` 用来验证按位与。  
这里 `0x30 & 0x0D = 0x00`，所以 `$5 = 0`。

### 5. `or`
`or $6,$1,$2` 用来验证按位或。  
这里结果是 `0x3D`。

### 6. `xor`
`xor $7,$1,$2` 用来验证按位异或。  
这里结果同样是 `0x3D`。

### 7. `sll`
`sll $8,$2,1` 用来验证逻辑左移。  
`13 << 1 = 26`，所以 `$8 = 0x1A`。

### 8. `srl`
`srl $9,$1,2` 用来验证逻辑右移。  
`48 >> 2 = 12`，所以 `$9 = 0x0C`。

### 9. `sra`
`sra $10,$1,2` 用来验证算术右移。  
这次被移位的是正数，所以结果和 `srl` 一样，都是 `12`。

### 10. `jr`
`jr $1` 是第一条控制流指令。  
因为 `$1` 里事先放的是 `0x30`，所以执行后 PC 会跳到地址 `0x30`，也就是索引 12。

### 11. `lui`
`lui $12,16'h00F0` 用来构造高位常数。  
执行后 `$12 = 0x00F00000`。

### 12. `ori`
`ori $14,$12,16'h00A5` 用来把低位常数拼进去。  
结果是 `$14 = 0x00F000A5`。

### 13. `andi`
`andi $13,$14,16'hF0F0` 用来验证逻辑立即数的零扩展。  
结果是 `$13 = 0x000000A0`。

### 14. `xori`
`xori $15,$14,16'h00FF` 用来翻转低 8 位。  
结果是 `$15 = 0x00F0005A`。

### 15. `sw`
`sw $13,8($17)` 用来验证存储器写入。  
这里真实地址是 `36 + 8 = 44`，不是模板中常见的 `0($0)` 特殊情况。

### 16. `lw`
`lw $18,8($17)` 用来从同一地址把数据读回。  
如果结果 `$18 = 0xA0`，说明地址计算和访存路径都没问题。

### 17. `beq`
`beq $13,$18,1` 用来验证相等分支。  
因为 `$13` 和 `$18` 相等，所以会跳过后面的 `nop`。

### 18. `bne`
`bne $15,$18,1` 用来验证不等分支。  
因为 `$15` 和 `$18` 不相等，所以也会跳过后面的 `nop`。

### 19. `j`
`j 0x2F` 是无条件跳转。  
它直接把 PC 改到后面的 `jal` 段。

### 20. `jal`
`jal 0x32` 除了跳转以外，还会把链接地址写到 `$31`。  
这条很适合讲“子程序调用”的概念：先保存返回点，再跳去目标地址。

## 六、12 条扩展整数指令讲解话术

### 1. `slt`
`slt $11,$2,$1` 的含义是：如果 `$2 < $1`，则目标寄存器写 `1`。  
这里 `13 < 48` 成立，所以 `$11 = 1`。

### 2. `bltz`
为了测试 `bltz`，我先构造了 `$22 = -5`。  
执行 `bltz $22,1` 时，由于 `$22 < 0`，所以分支成立，说明带符号比较路径是正确的。

### 3. `bgtz`
为了测试 `bgtz`，我构造了 `$23 = 7`。  
执行 `bgtz $23,1` 时，由于 `$23 > 0`，所以也会成功分支。

### 4. `mult`
`mult $2,$23` 用来做有符号乘法。  
这里 `13 * 7 = 91`，结果保存在 `HI/LO` 中，其中 `LO = 91`，`HI = 0`。

### 5. `mflo`
在 `mult` 后执行 `mflo $24`，就能把 `LO` 中的 `91` 读回到通用寄存器。  
这样可以直接在波形中验证乘法结果。

### 6. `mfhi`
紧接着用 `mfhi $25` 读出 `HI`。  
因为这次乘法结果没有超出 32 位，所以 `HI = 0`。

### 7. `multu`
`multu $1,$2` 用来做无符号乘法。  
这里 `48 * 13 = 624`，后面用 `mflo $26` 把结果读回。

### 8. `div`
`div $1,$2` 用来做有符号除法。  
这里 `48 / 13` 的商是 `3`，余数是 `9`，所以 `LO = 3`，`HI = 9`。

### 9. `divu`
`divu $1,$23` 用来做无符号除法。  
这里 `48 / 7` 的商是 `6`，余数是 `6`，后面分别通过 `mflo $29` 和 `mfhi $30` 读回。

### 10. `mthi`
`mthi $15` 的作用是把 `$15` 的值直接写入 `HI`。  
写完以后我再用 `mfhi $20` 读回，验证 `HI` 确实被改成了 `0x00F0005A`。

### 11. `mtlo`
`mtlo $13` 的作用是把 `$13` 的值直接写入 `LO`。  
写完以后用 `mflo $21` 读回，验证 `LO = 0xA0`。

### 12. `jalr`
`jalr $16,$19` 是扩展控制流指令。  
它和 `jal` 的区别在于：目标地址不是直接写在指令里，而是先放在寄存器 `$19` 中。  
执行后会跳转到 `$19` 指向的地址，同时把链接地址写到 `$16`。

## 七、为什么这版程序不会死循环
如果老师问“你怎么保证不会死循环”，我可以这样回答：
- `jr` 的目标是后面的有效指令区；
- `beq`、`bne`、`bltz`、`bgtz` 都只跳过单个 `nop`；
- `j` 直接跳到 `jal` 段；
- `jal` 再跳到 `jalr` 准备段；
- `jalr` 最终跳到最后一条结束标志指令；
- 测试平台只跑固定拍数，然后自动 `$finish`。  
所以整个流程是单向向前推进的，不会回跳到前面形成环路。

## 八、答辩时 3~5 分钟精简版讲稿
老师好，我这次完成的是一个基于 Verilog 的单周期 MIPS 模型机。和直接套用 demo 模板不同，我这次把测试参数、跳转地址和测试顺序都重新设计了一遍，目标是完整覆盖 **20 条 MIPS 基本整数指令** 和 **12 条 MIPS 扩展整数指令**，同时保证程序执行过程中不死循环、不跳飞、结果可直接在波形里验证。

整个模型机可以分成 5 个部分来理解：第一部分是 `IF`，负责维护 `PC` 并从 `InstMem` 取指；第二部分是译码和寄存器堆，负责拆分 `op/rs/rt/rd/shamt/imm` 并读出源操作数；第三部分是 `EX`，负责算术、逻辑、移位、访存地址计算以及 `HI/LO` 相关控制；第四部分是 `DataMem`，负责 `lw/sw`；第五部分是写回阶段，把计算结果、访存结果或者跳转链接地址写回寄存器。

我这次测试程序不是把所有指令直接堆在一起，而是按功能分段组织。前面先用 `addi` 构造常量，再依次验证 `add/sub/and/or/xor` 和 `sll/srl/sra`。然后通过 `jr` 跳过一个 `nop`，说明寄存器跳转通路是通的。接着我用 `lui/ori/andi/xori` 组合出特殊常量，再用 `sw/lw` 对同一地址先写后读，证明有效地址计算和访存通路正确。

在扩展部分，我先用 `beq`、`bne`、`bltz`、`bgtz` 验证条件分支，其中 `bltz` 用负数测试，`bgtz` 用正数测试，能体现带符号比较。然后我用 `mult/multu/div/divu` 改写 `HI/LO`，再通过 `mfhi/mflo` 读回结果；同时再用 `mthi/mtlo` 直接写 `HI/LO`，验证 `HI/LO` 的双向通路都没有问题。

最后一段我用 `j`、`jal`、`jalr` 串起三种典型跳转方式。`j` 是直接跳转，`jal` 会在跳转的同时保存链接地址，`jalr` 则是跳到寄存器给出的目标地址，并把链接地址写回指定寄存器。这样控制流、访存、ALU 和扩展通路就都覆盖到了。

这版程序之所以稳定，是因为我把所有跳转目标都设计成向后跳，所有条件分支都只跨过一个 `nop`，并且测试平台会在固定拍数后自动结束，所以不会形成死循环。答辩时如果看波形，我主要看 `pc`、`instruction`、`regaData/regbData`、`exOut`、`memOut`、`HI/LO` 以及关键寄存器的结果是否和调试表一致。这样就能证明这套 `20 + 12` 指令覆盖是完整而且正确的。

## 九、答辩时老师可能追问的问题与答法

### 问题 1：为什么你这次是 20 条基本指令加 12 条扩展指令？
答：因为课程设计要求完整覆盖 MIPS 的 20 条基本整数指令和 MIPS12 条扩展整数指令，所以我把测试程序调整成了 `20 + 12` 的完整版本，而不是只演示部分扩展指令。

### 问题 2：你怎么证明 `bltz` 和 `bgtz` 真的是按有符号数判断的？
答：我分别构造了负数 `$22 = -5` 和正数 `$23 = 7`，然后让 `bltz` 和 `bgtz` 各自分支一次，这样可以在波形里直接看到两个条件分支都按预期成立。

### 问题 3：你怎么证明 `andi/ori/xori` 真的是零扩展？
答：我专门用了 `16'hF0F0` 和 `16'h00FF` 这种高位不为 0 的立即数。尤其是 `andi` 最终得到 `0x000000A0`，如果还是符号扩展，就不会得到这个结果。

### 问题 4：你怎么证明 `lw/sw` 不是只在 `0($0)` 这种特殊情况下正确？
答：我这里用的是 `8($17)`，其中 `$17 = 36`，所以真实访问地址是 `44`。这证明我测试的是“基址 + 偏移”的一般情况。

### 问题 5：你怎么证明 HI/LO 通路是正确的？
答：我用了两种方法：第一种是 `mult/div` 之后立刻用 `mfhi/mflo` 读回；第二种是直接用 `mthi/mtlo` 改写 `HI/LO`，再用 `mfhi/mflo` 读回验证。这样 HI/LO 的读写两条路径都测到了。

### 问题 6：为什么 `jal` 和 `jalr` 写回的是下一条地址？
答：因为本工程的 `PC` 和 `InstMem` 都是同步时序结构，执行跳转链接指令时，顶层 `pc` 线上看到的已经是下一条地址，所以写回的链接值表现为“当前逻辑指令地址 + 4”。
